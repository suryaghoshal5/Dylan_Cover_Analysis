🎸 Dylan Analysis Workflow

  1. Find Bob Dylan
    SELECT id FROM artist WHERE name = 'Bob Dylan'
  2. Get Dylan's Works
    SELECT w.* FROM work w JOIN l_artist_work law ON w.id = law.work WHERE law.artist = [dylan_id]
  3. Find All Recordings (No Cap!)
    SELECT r.* FROM recording r JOIN recording_work rw ON r.id = rw.recording WHERE rw.work_id = [work_id]
  4. Identify Covers
    Filter: artist_credit != Bob Dylan
  5. Enrich with Spotify
    Match via ISRCs for popularity scores
  6. Analyze Patterns
    Most covered songs, genre evolution, temporal trends

🤖 SLM Training Workflow

  1. Build Artist Graph
    Extract all l_artist_* relationships → NetworkX graph
  2. Create Work→Recording Graph
    Cover relationships as directed edges
  3. Generate Entity Embeddings
    Node2Vec on artist/work/recording graphs
  4. Extract Tag Co-occurrences
    Build genre similarity matrix from tag overlap
  5. Temporal Feature Extraction
    Career spans, release patterns, genre evolution
  6. Multi-Modal Training Corpus
    Combine: metadata + annotations + relationships + embeddings

🤖 Music SLM/LLM Training Considerations

1. Knowledge Graph Structure

Node Type Entity Count Embedding Dimension Training Use
Artist ~2M 256-512 Artist similarity, influence propagation
Work ~2.5M 256-512 Composition similarity, cover prediction
Recording ~30M 128-256 Performance style, acoustic similarity proxy
Release ~3M 128 Album context, temporal patterns
Tag ~50K 64-128 Genre/mood classification, style transfer

2. Relationship Types for Graph Learning

Relationship Edge Count Directionality Learning Task
artist →[composed]→ work ~2.5M Directed Composition attribution, artist style
work →[recorded_as]→ recording ~30M Directed Cover prediction, version clustering
artist →[performed]→ recording ~30M Directed Performance attribution, cover detection
artist →[collaborated_with]→ artist ~500K Undirected Collaboration prediction, band formation
recording →[tagged_as]→ tag ~10M Directed Genre classification, mood prediction

3. Training Data Exports

Recommended Export Formats:
  • JSON-LD: Linked data for knowledge graph reasoning
  • TSV Triples: Subject-Predicate-Object for graph embeddings
  • Parquet: Efficient columnar format for large-scale training
  • NetworkX Graph: Python graph object for rapid prototyping

4. Potential SLM Training Tasks

Task Input Output Evaluation Metric
Cover Prediction Work + Artist embeddings Likelihood of cover existing Precision@K, Recall@K
Genre Classification Recording metadata + tags Multi-label genre predictions F1-score (micro/macro)
Artist Influence Artist graph topology Influence scores (PageRank-style) Ground truth: cover frequency
Temporal Music Trends Release dates + genre tags Genre popularity over time Forecasting accuracy
Collaboration Prediction Artist embeddings + past collabs Future collaboration likelihood Link prediction AUC
Music Question Answering Natural language query Entity retrieval from graph MRR, Hits@1/10

✅ Post-Import Validation Queries

BOTH Basic Sanity Checks

Check SQL Query Expected Result
Total artists SELECT COUNT(*) FROM artist; ~2,000,000
Total works SELECT COUNT(*) FROM work; ~2,500,000
Total recordings SELECT COUNT(*) FROM recording; ~30,000,000

DYLAN Dylan-Specific Validation

Check SQL Query Expected Result
Bob Dylan exists SELECT * FROM artist WHERE name = 'Bob Dylan'; 1 row (artist ID)
Dylan works SELECT COUNT(*) FROM l_artist_work WHERE artist = [dylan_id]; ~650 works
"Blowin' in the Wind" covers SELECT COUNT(*) FROM recording r JOIN recording_work rw ON r.id = rw.recording JOIN work w ON rw.work_id = w.id WHERE w.name LIKE '%Blowin%Wind%'; 400+ (NOT capped at 100!)

SLM Graph Completeness Checks

Check SQL Query Purpose
Artist→Work edges SELECT COUNT(*) FROM l_artist_work; Composition graph density
Work→Recording edges SELECT COUNT(*) FROM recording_work; Cover graph density
Artist→Artist edges SELECT COUNT(*) FROM l_artist_artist; Collaboration graph density
Tags per recording (avg) SELECT AVG(tag_count) FROM (SELECT recording, COUNT(*) AS tag_count FROM recording_tag GROUP BY recording); Genre label richness
Most tagged genres SELECT t.name, COUNT(*) FROM tag t JOIN recording_tag rt ON t.id = rt.tag GROUP BY t.name ORDER BY COUNT(*) DESC LIMIT 20; Genre distribution baseline
