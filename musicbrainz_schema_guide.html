3. mbdump-edit.tar.bz2 - Edit History

Size: ~14.6 GB | Contains: Complete edit history and voting records

Purpose: Tracks every change ever made to the database. Largest file but least relevant for cover analysis. Mostly useful for data quality auditing.

Table Name Purpose
edit Every modification made to the database (create, update, delete operations)
edit_artist Artist edits (e.g., when "Bob Dylan" entry was created/modified)
edit_work Work/composition edits (when Dylan songs were added)

⚠️ Note: We don't need most of this data for Dylan analysis. The edit history is valuable for MusicBrainz's operations but not for our cover research. We import it for database completeness.

🔍 Query Workflow for Dylan Cover Analysis

How we'll use these tables to extract Dylan's covers:

  1. Find Bob Dylan's Artist ID
    SELECT id FROM artist WHERE name = 'Bob Dylan'
  2. Find All Works by Bob Dylan
    SELECT w.* FROM work w JOIN l_artist_work law ON w.id = law.work WHERE law.artist = [dylan_id]
  3. Find All Recordings of Each Work
    SELECT r.* FROM recording r JOIN recording_work rw ON r.id = rw.recording WHERE rw.work_id = [work_id]
  4. Identify Covers (Non-Dylan Recordings)
    SELECT r.* FROM recording r JOIN artist_credit ac ON r.artist_credit = ac.id JOIN artist_credit_name acn ON ac.id = acn.artist_credit WHERE acn.artist != [dylan_id]
  5. Get Release Info for Each Cover
    SELECT rel.* FROM release rel JOIN medium m ON rel.id = m.release JOIN track t ON m.id = t.medium WHERE t.recording = [recording_id]
  6. Get Release Dates
    SELECT * FROM release_country WHERE release = [release_id] ORDER BY date_year, date_month, date_day LIMIT 1
  7. Enrich with Spotify (Post-Processing)
    Use ISRCs from the isrc table to match recordings with Spotify's API for popularity scores.
